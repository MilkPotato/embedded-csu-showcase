## 一、项目概述

### 1.1 项目定位

本系统是一个**纯静态课程成果展示网站**，用于集中展示中南大学自动化学院"微控制器与嵌入式系统课程设计"的优秀学生作品。网站集课程信息发布、优秀作品展陈、教学资源下载于一体，面向历届学生开放浏览。

### 1.2 核心目标

1. **教学成果展示** — 将学生作品以标准化的卡片形式呈现，便于下一届借鉴参考
2. **课程信息公开** — 公布评分标准、平台选型指南，提升教学透明度
3. **资源沉淀共享** — 汇集任务书、FAQ、工具链配置等教学资源
4. **隐私与版权保护** — 通过脱敏、密码门禁、爬虫屏蔽等手段保护学生隐私

### 1.3 技术特性

| 特性 | 说明 |
|------|------|
| 架构模式 | 零构建静态单页应用（SPA） |
| 托管平台 | Netlify + GitHub |
| 部署方式 | CI/CD：Git push → 自动部署 |
| 存储策略 | 代码在 GitHub，视频在 B站，图片本地 |
| 统计数据 | 百度统计 + 不蒜子 |
| 访问控制 | 前端密码门禁 + robots.txt + noindex |

## 五、核心功能模块

### 5.1 密码门禁

- **实现**：页面加载时全屏覆盖层，输入密码后隐藏
- **密码存储**：明文硬编码于 JS 中 `ACCESS_PASSWORD` 变量（当前：206207208）
- **状态保持**：`sessionStorage` — 同标签页内刷新无需重复输入
- **安全等级**：前端防君子不防小人（可被开发者工具绕过）
- **辅助防护**：`<meta robots="noindex, nofollow">` + `robots.txt` 防搜索引擎

### 5.2 课程概览

- 四张信息卡片：课程名称 / 面向班级 / 指导教师 / 时间
- 设计目的说明
- MCU vs MPU 双阵营对比卡片
- 六项评分标准表格（权重+说明）

### 5.3 作品展示（核心）

**筛选系统**（三重筛选，可组合）：

| 筛选项 | 类型 | 数据来源 |
|--------|------|----------|
| 分类筛选 | 按钮组切换 | `CATEGORIES` 常量 |
| 年份筛选 | 标签页切换 | `work.year` 字段（自动提取去重） |
| 关键词搜索 | 实时输入 | `work.students / title / class / tags` |

**卡片渲染**：

```
┌───────────────────────┐
│  [封面图 / 渐变背景]   │  ← 有 B站链接 → B站封面图 + 播放按钮覆盖层
│                       │  ← 无 B站链接 → 分类渐变背景 + 图标
│  分类标签    亮点标签   │
├───────────────────────┤
│  作品标题               │
│  👤 作者姓名            │
│  📚 班级 · 年级 · 年份  │
│  项目描述（最多3行）     │
│  [STM32] [游戏] [LCD]  │  ← 技术标签
│  [📂代码] [📄报告]     │  ← 操作按钮
└───────────────────────┘
```

**封面图策略**：
- 如果 `work.videoUrl` 包含 B站链接 → 提取 BV 号 → 加载 `assets/covers/{BV号}.jpg`
- 加载成功显示图片 + 悬浮播放按钮 → 点击跳转 B站
- 加载失败或无 B站链接 → 显示分类默认渐变色 + 图标

### 5.4 资源支持

- **任务书下载**：三份文档入口（待上传时显示"待上传"状态）
- **FAQ**：手风琴式折叠，点击展开/收起
- **工具链**：6 个常用开发工具卡片，点击跳转官网

### 5.5 统计分析

| 系统 | 用途 | 部署方式 |
|------|------|----------|
| 百度统计 | 详细访问分析（地域、趋势、设备） | `<head>` 中注入代码 |
| 不蒜子 | 简易访客计数（UV + PV） | 页脚嵌入标签 |

## 六、视觉设计

### 6.1 设计风格

- **主色调**：深蓝 (`#1a3a6b`) — 匹配中南大学品牌色
- **强调色**：红色 (`#c41a1a`) — 按钮、权重数字
- **布局**：单栏居中，最大宽度 1280px
- **字体**：系统原生字体栈，优先 PingFang SC

### 6.2 响应式断点

| 断点 | 适配 |
|------|------|
| `>768px` | 桌面：多列卡片网格，水平导航 |
| `≤768px` | 平板：汉堡菜单，单列/双列调整 |
| `≤480px` | 手机：全单列，紧凑间距 |

## 七、部署与运维

### 7.1 部署流水线

```
本地修改 (data.js / config.js)
        ↓
git add + commit + push origin master
        ↓
GitHub master 分支已更新（不触发部署）
        ↓
GitHub PR: master → production
        ↓
git push origin production 或 Merge PR
        ↓
Netlify 检测到 production 更新
        ↓
Netlify 构建（复制 docs/ 目录）
        ↓
CDN 全球分发上线
```

## 八、隐私与安全措施

| 措施 | 实现方式 | 效果 |
|------|----------|:----:|
| 姓名脱敏 | 姓名第2字 → `*` | ✅ 降低可识别性 |
| 班级脱敏 | `自动化2401` → `自动化240*班` | ✅ 模糊具体班级 |
| 搜索屏蔽 | `noindex, nofollow` + `robots.txt` | ✅ 正规爬虫不收录 |
| 引用隐藏 | `<meta referrer="no-referrer">` | ✅ 浏览器不发送 Referer |
| 访问门槛 | JS 密码门禁 | ✅ 防止随意查看 |
| 非官方声明 | 页脚显示免责声明 | ✅ 法律声明 |

## 十一、项目风险与注意事项

1. **密码安全性**：前端硬编码密码仅防随手翻阅，不能替代后端认证
2. **B站链接有效性**：视频被删除或设为私密会导致封面跳转 404
3. **百度统计依赖网络**：国内用户正常加载，国外可能被拦截
4. **Netlify 免费额度**：每月 300 积分限制，高峰期注意规划
5. **GitHub 国内访问**：如遇封锁，可通过 Netlify 直接部署（无需 GitHub 访问）
